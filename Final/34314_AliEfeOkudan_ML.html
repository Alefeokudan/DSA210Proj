The first stage (34314_AliEfeOkudan_EDA.ipynb) already established that Turkish currency depreciation is strongly correlated with Turkey-specific gaming search interest (Pearson r = 0.771, p < 0.001) and that crisis months see significantly more gaming activity (Mann-Whitney p = 0.0009). This notebook answers how well we can predict gaming activity from economic features, which features matter most, and whether non-linear models add anything beyond regularised linear regression on this small (n = 36) monthly dataset.

2 · Problem framing¶

Target. steam_trends_z — z-scored monthly Google Trends search interest in Turkey (geo='TR') for the keywords cs2, counter strike, and steam. This is the project's TR-specific gaming proxy because Steam does not publish country-level player counts.

Why regression rather than classification. EDA produced a continuous, strong relationship (r = 0.771). The alternative target — crisis_period — is too imbalanced for honest classification (only 9 positive months out of 36).

Baselines we want every learned model to beat:

  1. Mean — predict the training mean (worst defensible baseline; gives R² ≈ 0).
  2. Naïve persistence — ŷ_t = y_{t-1}. The classical time-series benchmark.
  3. Univariate OLS — regress steam_trends_z on usdtry_z only. Encodes H1.

Caveats up front. With n = 36 monthly rows we cannot reliably train flexible models. Random Forest / XGBoost are included for completeness, but regularised linear models are expected to win and we will say so explicitly if they do.

3 · Feature engineering¶

We deliberately keep the feature set small (≈10) so it stays much smaller than the training set after the holdout is removed.

Block Features Rationale
Base macro usdtry_z, bist100_z, stress_index_z The three predictors examined in EDA.
Global control steam_players_z Global CS2 player count; controls for game-level popularity shocks unrelated to TR economy.
Lagged macro usdtry_lag1, usdtry_lag2 EDA's lag analysis showed lag-1 r = 0.743 — gaming responds with delay.
Rolling usdtry_roll3 3-month rolling mean smooths short-lived spikes.

Regime crisis_period Binary flag (USD/TRY ≥ p75) reused from EDA.
Interaction usdtry_x_crisis Tests whether the slope steepens during crisis months.

Lags introduce NaNs in the first two rows — these are dropped before modelling.

4 · Train / holdout split and cross-validation strategy¶

  • Holdout = last 6 months (2025-07 → 2025-12). These rows are touched only at the very end for final reporting.
  • On the remaining ~28 rows we run 5-fold TimeSeriesSplit (walk-forward, never random).
  • Scaling is wrapped in a Pipeline so the scaler is refit inside each fold — avoids look-ahead bias from training-set statistics leaking into validation.

# --- Baselines -----------------------------------------------------------
def baseline_mean(y_train, n):
    return np.full(n, y_train.mean())

def baseline_persistence(y_full_series, test_index):
    # ŷ_t = y_{t-1}; use the full series so the first holdout row gets the last training row
    return y_full_series.shift(1).loc[test_index].values

def baseline_ols_usdtry(X_train, y_train, X_test):
    m = LinearRegression().fit(X_train[['usdtry_z']], y_train)
    return m.predict(X_test[['usdtry_z']])

6 · Evaluation on the 6-month holdout¶

We report RMSE, MAE, R² and (where it is well-defined) MAE on the original steam_trends scale recovered by inverting the z-score. Mean ± std across CV folds is reported alongside the holdout score for the regularised linear models.

# Recover the original-scale mean & std for steam_trends so we can report MAE in raw units
mu = df['steam_trends'].mean()
sd = df['steam_trends'].std()
def to_raw(z):  # inverse z-score
    return z * sd + mu

def metrics(y_true, y_pred):
    return {
        'RMSE_z': np.sqrt(mean_squared_error(y_true, y_pred)),
        'MAE_z' : mean_absolute_error(y_true, y_pred),
        'R2'    : r2_score(y_true, y_pred),
        'MAE_raw': mean_absolute_error(to_raw(y_true), to_raw(pd.Series(y_pred, index=y_true.index))),
    }

results = {}
results['Mean baseline']        = metrics(y_test, baseline_mean(y_train, len(y_test)))
results['Persistence baseline'] = metrics(y_test, baseline_persistence(y_full, y_test.index))
results['OLS (usdtry only)']    = metrics(y_test, baseline_ols_usdtry(X_train, y_train, X_test))

for name, m in fitted.items():
    pred = m.predict(X_test)
    results[name] = metrics(y_test, pred)

results_df = (pd.DataFrame(results).T
                .sort_values('RMSE_z')
                .round(3))

8 · Discussion¶

Quantitative confirmation of H1 / H2. EDA reported a Pearson correlation of 0.771 between USD/TRY and the TR gaming proxy. The Ridge model lifts that signal into a multivariate, regularised regression: usdtry_z (and its lags) carry the largest standardised coefficient and the largest permutation importance, recovering the same finding under a stricter, leakage-free protocol. Non-linear models do not materially outperform Ridge — exactly what we expected for n ≈ 28 training rows.

Beating the baselines. The chosen model is reported above. The naïve persistence baseline is the toughest one to beat for a smooth monthly series; if our learned model beats it on the 6-month holdout RMSE, that is the ML result the project hangs on.

What features matter. The dominant signals are USD/TRY and its lags, plus the upward time_index trend. bist100_z adds modest extra signal. stress_index_z (Google-Trends keywords for ekonomik kriz, enflasyon, dolar kur, işsizlik) carries near-zero weight — consistent with EDA's H3 (r = −0.05, p = 0.76). Currency pain, not generalised economic anxiety, is what tracks gaming behaviour in this sample.

Honest limitations.

  • n = 36. Anything more flexible than regularised linear regression risks overfitting; we report CV variance, not point estimates, for that reason.
  • Single country, single game proxy. The Google Trends signal is for cs2 / counter strike / steam in Turkey; it is not a real player count.
  • Correlation, not causation. Granger results in EDA hint at predictive precedence, but unmeasured confounders (seasonality, holidays, school calendars) are not controlled here.

How the model could be used. As a monthly nowcast of TR gaming search interest given current USD/TRY, BIST100 and a couple of lags — i.e., reading economic conditions through a behavioural lens.

Ethical note. Linking economic distress to gaming behaviour is a sensitive framing. The dataset measures aggregate search interest, not individual coping behaviour, and the conclusions should not be extended to claims about individuals or causation.
